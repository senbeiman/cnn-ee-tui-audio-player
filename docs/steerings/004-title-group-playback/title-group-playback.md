# タイトル単位再生

## 概要
英語学習用の音声ファイルは、同じタイトルの複数トラックが並ぶことがある。アプリの再生操作は、選択中のトラックから同じタイトルの範囲だけを続けて再生できれば十分とする。

## 要件

### 基本機能
- `p`キーで選択中MP3を再生開始する
- 再生開始時のタイトルを保持する
- トラック終了時、次のMP3のタイトルが開始時と同じなら自動再生する
- 次のMP3のタイトルが変わった場合は自動停止する
- 次のMP3が存在しない場合は自動停止する
- ディレクトリは自動遷移対象にしない

### キーバインディング
- **p**: 選択中MP3からタイトル単位再生開始
- **Space**: 一時停止/再開
- **Enter**: ディレクトリ選択・移動
- **j/k**: ファイル選択（再生は継続）
- **s/r/c**: 再生機能としては使用しない

### タイトル判定
ファイル名ベースで同一タイトルを判定する。先頭の番号プレフィックスだけを除外し、それ以外は拡張子を含むファイル名の完全一致で判定する。

1. ファイル名先頭にASCII数字が連続し、その直後に空白がある場合は、数字と空白を除く
2. 除外後のファイル名を完全一致で比較する
3. 括弧内や末尾番号など、それ以外の正規化は行わない

例:
- `02　Cinema Update.mp3` → `Cinema Update.mp3`
- `03　Cinema Update.mp3` → `Cinema Update.mp3`
- `基礎トレーニング1(英→日).mp3` と `基礎トレーニング1(日→英).mp3` は別タイトルとして扱う
- `01　基礎トレーニング1(英→日) 1.mp3` と `02　基礎トレーニング1(日→英).mp3` は別タイトルとして扱う

## 実装方針

### 1. 再生モードの整理
再生モード分岐は削除し、`Player` は選択された1トラックを1回だけ再生する責務に戻す。リピート再生と全MP3連続再生は削除する。

### 2. App側の状態
再生開始時のタイトルを `App` に保持し、自動遷移判定に使う。

```rust
pub struct App {
    playback_title: Option<String>,
}
```

### 3. 自動遷移
`update()` はトラック終了を検知したら、現在再生中ファイルの次のMP3を確認する。次のMP3のタイトルが開始時のタイトルと一致する場合だけ再生し、一致しない場合は停止する。

## 動作シナリオ

### 正常系
1. `02　Cinema Update.mp3` を選択して `p` 押下
2. トラック終了時、次のMP3のタイトルを確認
3. 次のMP3が `03　Cinema Update.mp3` なら、先頭番号を除いたタイトルが同じなので自動再生
4. 次のMP3が `04　特集.mp3` ならタイトルが変わるため停止

### 境界条件
- タイトル内の最後のトラックから開始した場合は、そのトラック終了時に停止
- ディレクトリ移動時は現在の再生を停止しないが、ファイルリストが変わるため自動遷移対象はなくなる
- MP3以外のファイルはリストにも自動遷移にも含めない
