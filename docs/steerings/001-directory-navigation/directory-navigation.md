# ディレクトリナビゲーション機能仕様

## 概要
TUI Media Playerにディレクトリナビゲーション機能を追加し、ユーザーがディレクトリ階層を移動してファイルを選択できるようにする。

## 現状の課題
- 現在は起動時に指定されたディレクトリ以下のMP3ファイルを再帰的に全て表示
- ディレクトリ構造が見えず、ファイル管理が困難
- 特定のサブディレクトリ内のファイルのみを再生したい場合に不便

## 機能要件

### 1. ディレクトリ・ファイル混合表示
- 現在のディレクトリ内のサブディレクトリとMP3ファイルを同じリストに表示
- ディレクトリは名前の前に `📁` アイコンを付けて区別
- MP3ファイルはプレフィックスなしで表示

### 2. 上位ディレクトリへの移動
- Escキーで一つ上のディレクトリに移動
- ルートディレクトリ（設定された最上位）に到達した場合はEscキーで移動不可

### 3. 下位ディレクトリへの移動
- ディレクトリエントリを選択（Space）すると、そのディレクトリに移動
- 移動後はそのディレクトリ内のコンテンツを表示

### 4. ファイル再生
- MP3ファイルを選択（Space）すると再生開始
- 既存の再生/一時停止機能と統合

## 技術要件

### 1. データ構造の拡張
```rust
#[derive(Debug, Clone)]
pub enum FileType {
    Directory,
    Mp3File,
}

#[derive(Debug, Clone)]
pub struct FileInfo {
    pub path: PathBuf,
    pub name: String,
    pub file_type: FileType,
}
```

### 2. アプリケーション状態の拡張
```rust
pub struct App {
    // 既存フィールド
    pub files: Vec<FileInfo>,
    pub selected: usize,
    pub scroll_offset: usize,
    pub player: Player,
    pub should_quit: bool,

    // 新規フィールド
    pub current_directory: PathBuf,
    pub root_directory: PathBuf,  // 移動可能な最上位ディレクトリ
}
```

### 3. ファイルスキャン機能の変更
- `scan_mp3_files()` を `scan_current_directory()` に変更
- 再帰的スキャンではなく、現在ディレクトリ内のみをスキャン
- ディレクトリとMP3ファイルの両方を返す

## UI表示の変更

### 1. ファイルリスト表示
```
┌─ファイルリスト (2/7) ─┐
│> 📁 01_basics          │
│  📁 02_intermediate    │
│  audio1.mp3            │
│  audio2.mp3            │
│  ...                   │
└────────────────────────┘
```

### 2. タイトルバーにディレクトリパス表示
- 現在のディレクトリパスを表示（相対パスまたは短縮形）

## 操作フロー

### 1. ディレクトリ移動
1. j/k でディレクトリエントリを選択
2. Space で選択 → ディレクトリに移動
3. ファイルリストが新しいディレクトリの内容で更新
4. 選択位置は先頭に初期化

### 2. 上位ディレクトリ移動
1. Escキー押下 → 上位ディレクトリに移動
2. 移動元ディレクトリにカーソルが自動選択
3. ルートディレクトリの場合は移動しない

### 3. ファイル再生
1. j/k でMP3ファイルを選択
2. Space で選択 → 再生開始/一時停止

## エラーハンドリング

### 1. 権限エラー
- ディレクトリにアクセス権限がない場合は移動しない
- エラーメッセージを一時表示

### 2. 空ディレクトリ
- MP3ファイルもサブディレクトリもない場合は "ファイルなし" メッセージを表示
- Escキーでの上位移動は引き続き可能

## 下位互換性
- 既存の起動方法（ディレクトリ指定）は継続サポート
- 指定されたディレクトリを初期ディレクトリ兼ルートディレクトリとして設定
